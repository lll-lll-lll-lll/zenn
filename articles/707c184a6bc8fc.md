# はじめに
以下のケースはよくあるのではないでしょうか？
「ユーザーのメールアドレスは任意入力だが、入力される場合は重複させたくない」というよくある要件。。
ごく自然に ALTER TABLE users ADD UNIQUE (email); のようなDDLを実行する。
しかし、実際にemailカラムに複数のNULLが登録できてしまい、「UNIQUE制約を付けたはずなのに、なぜ？」となる
